Recursive-descent / Predictive Parsing

works only on grammars where the first terminal symbol of each subexpression provides enough information to choose which production to use.

First and Follow Set

Given a string \(\gamma\) of terminal and nonterminal symbols, \(FIRST(\gamma)\) is the set of all terminal symbols that can begin any string derived from \(\gamma\).

If two different productions \(X \rightarrow γ1\) and \(X \rightarrow γ2\) have the same left-hand side symbol \(X\) and their right-hand sides have overlapping FIRST sets, then the grammar cannot be parsed using predictive parsing. If some terminal symbol \(I\) is in \(FIRST(\gamma1)\) and also in \(FIRST(\gamma2)\), then the X function in a recursive-descent parser will not know what to do if the input token is \(I\) .

Symbols can produce the empty string is \(nullable\).

\(FOLLOW(X)\) is the set of terminals that can immediately follow \(X\). That is, \(t \in FOLLOW(X)\) if there is any derivation containing \(Xt\). This can occur if the derivation contains \(XYZt\) where Y and Z both derive \(\epsilon\).

To construct the predictive parsing table, enter production \(X \rightarrow γ\) in row \(X\), column \(T\) of the table for each \(T \in FIRST(\gamma)\). Also, if \(\gamma\) is nullable, enter the production in row \(X\), column \(T\) for each \(T \in FOLLOW[X]\)

An ambiguous grammar will always lead to duplicate entries in a predictive parsing table.

\(LL(1)\) is a non-ambiguous grammar, short for Left-to-right parse, Leftmost-derivation, 1-symbol lookahead.

To make a language \(LL(1)\), left recursion should be removed.

\(LR(1)\)的parsing table会很大，所以往往会采用相对弱一些的\(LALR\).

相比较\(LR(1)\), \(LALR\)将只有lookahead不同的状态合并成一个状态。

状态的合并可能引入reduce-reduce conflict。

右边的状态在遇到a或b时会不知道对规则\(A\rightarrow\alpha\)还是\(B\rightarrow\beta\)进行规约。

\(LALR\)在编译中最为常用。

如果用LR parsing处理unambiguous grammar \[ \begin{align} &S\rightarrow \text{if E then S else S}\\ &S\rightarrow \text{if E then S}\\ &S\rightarrow other \end{align} \] 会存在shift-reduce conflict，可以将grammar改写成 \[ \begin {align} &S\rightarrow M\\ &S\rightarrow U\\ &M\rightarrow \text{if E then M else M}\\ &M\rightarrow \text{other}\\ &U\rightarrow \text{if E then S}\\ &U\rightarrow \text{if E then M else U} \end{align} \] 但这样会引入较多的状态，往往采用优先shift的策略。
